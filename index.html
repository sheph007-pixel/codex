Email Throttle

Limit outbound emails from this system to prevent accidental mass sends and stay within provider limits.

Microsoft 365 allows 30 messages/min. Keep well under this to avoid temporary blocks.
Exchange Online caps recipients at 10,000/day. A conservative daily cap protects deliverability and sender reputation.
Adds a cooldown between consecutive sends. Prevents burst sending that triggers spam filters and Graph API throttling (429 errors).
Recommended Settings
Daily limit: 200/day — stays well under Exchange Online's 10,000 recipient cap and protects sender reputation Hourly limit: 20/hr — well within Microsoft 365's 30 msgs/min, prevents burst throttling Min delay: 60s — one email per minute paces sends naturally and avoids spam filter triggers
Microsoft 365 hard limits: 30 msgs/min, 10,000 recipients/day. Staying well below these protects your sender reputation and avoids temporary blocks.
Limits reset on a rolling window. Leave blank to disable throttling.

Azure App Configuration

To connect your Outlook, you need a Microsoft Azure App Registration. This gives Kennion CRM permission to read your emails.

  1. Go to portal.azure.com → Microsoft Entra ID → App registrations
  2. Click New registration, name it "Kennion CRM"
  3. Set Redirect URI (SPA) to: https://sheph007-pixel.github.io/codex/
  4. Under API permissions, add: Mail.Read, Mail.Send, and Files.ReadWrite.AppFolder (Delegated)
  5. Click Grant admin consent (or have your admin do it)
  6. Copy the Application (client) ID and paste it below

This is stored securely in your cloud database.
